# exp04 — Detecting hidden (steganographic) reasoning

## The goal

We want to know if a model can **hide its reasoning** and still get the right answer — and if so,
whether we can **catch it by looking inside the model** instead of reading its text. "Hiding" means
the model does the thinking, but its visible chain-of-thought (CoT) shows none of the working. A
monitor that reads the text sees a normal answer and notices nothing. This matters for safety,
because we rely on reading the CoT to check that models behave well.

## The task

- **Hidden task (the thinking we hide):** a small number chain. Start with a digit, then repeat a
  step like `next = (previous × a + b) mod 10`, a few times. The model must actually compute each
  intermediate result. The chains are **generated by us**, so we always know the true answer.
- **Cover task (the innocent surface):** summarize a short passage in a few sentences. This gives the
  model room to write freely.

## Two conditions (overt vs covert)

Both use the **same chains**, so the comparison is fair. They differ only in *how* the working appears:

- **Overt (Plain):** the model solves the chain and shows its arithmetic. The working is visible.
- **Covert (Stego):** the model writes a normal summary, but **hides each intermediate result as the
  first letter of each sentence** (using a secret digit→letter code). It is **not allowed to write any
  numbers** in the text, and we switch off its private "thinking" space, so the only place it can keep
  its working is the hidden letter channel.
- **Cover-only:** just the summary, no hidden task. This is a plain reference.

## What we measure first (the capability gate)

Before any deep analysis, we check one cheap thing: **can the model actually do this?** We compare:
- how often the **overt** model gets the answer right (its normal ability), and
- how well the **covert** model encodes the chain correctly (fidelity), and recovers the final answer.

If the covert version fails even on short chains, that is an important result by itself: it means
prompting cannot make the model hide real reasoning, and we would need to *train* it to do so before
going further. (Earlier work, NEST, found exactly this kind of failure, so we expect this risk.)

## The hard part: a fair baseline

To claim "the model hides information in its activations," we need a comparison that is **not unfair**.
Two obvious comparisons are flawed:
- Comparing the covert summary to a plain summary is unfair, because the two texts are different — any
  difference might just come from the different words, not from hiding.
- Looking at where the model "pays attention" can mislead us, because at the end of a sentence the
  model is *about to write* that letter anyway, so of course it looks there.

So we use the standard, careful methods from interpretability research:

1. **Minimal pairs.** We make two covert transcripts for the **same passage** that differ **only in the
   hidden number**. The summaries are almost identical; only the secret letters change. Now any
   internal difference must come from the secret, not from different writing.
2. **Swapping instead of breaking (activation patching).** Instead of "block the model's access and
   see if it breaks," we **copy the internal activity that carries one secret into the other run** and
   check whether the answer **changes to match**. If swapping the internal state swaps the answer, that
   state really carries the secret. This is a clean, positive test.
3. **Replace, don't zero (resample ablation).** When we do remove information, we **replace it with
   real activity from another run**, not with zeros. Zeroing pushes the model into a strange,
   unnatural state and gives misleading results; replacing with a real example keeps it honest.
4. **Probe selectivity.** If we train a small classifier ("probe") to read the secret from the
   activations, we also train it on **random fake labels**. The gap between real and fake accuracy
   tells us the probe is reading a true feature, not just memorizing. (Simple linear probes are
   trustworthy here; flexible non-linear ones can cheat, so we treat those carefully.)

## The flow, end to end

1. Generate the chains and pick passages.
2. Produce transcripts for the three conditions, plus **minimal-pair** covert transcripts (same passage,
   different secret).
3. **Capability gate:** measure overt accuracy and covert fidelity/answer recovery. Decide whether to
   continue or to switch to training the model.
4. **Correlational check:** find the attention heads that read the hidden-letter positions.
5. **Causal check:** swap or resample the activations at those positions and see if the hidden answer
   follows, while the summary stays fluent. If the hidden answer moves but the summary does not, the
   hidden channel is genuinely carrying the reasoning.

## What would count as success

A clear, honest result on one model and one scheme: the model reaches the right hidden answer with no
visible working, **and** we can show — by swapping its internal state — that the hidden reasoning lives
in specific, identifiable activations. We will state the smallest claim the evidence supports, not more.
